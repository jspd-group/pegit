Help Manual -COMMIT COMMAND

Name

peg-commit - Permanently record changes to the repository and store them in the permanent database.

Syntax

peg commit [-m][--message][-d][--description][-t][--tag]
[commit_message | description | tag_value]

Description

Peg commit command - This command stores the current contents of the cache / temporary database (which was stored using peg insert command ) in a new commit along with a log message from the user describing the changes.
The content to be added can be specified in several ways:
1. By using peg insert to incremently add changes to temporary cache before using peg commit command
2. By using peg reset command to remove files from temporary cache.
Finally we use peg commit command to store the changes present in the temporary database/cache to store them as a new commit with a given message or a given tag.
The peg status command can be used to obtain a summary of which files have changes that are staged for the next commit i.e. which files have been added to temporary database which are yet to be committed .

Options

  • [-m] or [--message] "message" : This option uses the given "message" as the commit message. It is often beneficial to add a commit message as it helps in easy debugging and also helps in remembering what changes were made in a paticular commit
  • [-d] or [--description] "description": This option is used to add a description alongwith every commit made . This option is followed by a description
  • [-t] or [--tag] : Often we reach a point , where we can be sure of giving a new version to our project. In this case we provide a tag to the present state of the project .

Examples

  • When recording your own work, the contents of modified files in your working tree are temporarily stored to a staging area called the cache / temporary database with peg insert. A file can be reverted back, only in the index but not in the working tree, to that of the last commit with peg reset command, which effectively reverts peg insert and prevents the changes to this file from participating in the next commit. After building the state to be committed incrementally with these commands, peg commit (without any pathname parameter) is used to record what has been staged so far. This is the most basic form of the command. An example:
    •  $ peg insert given_file.txt 
 # This command will insert given_file.txt into cache
 $ peg reset 
 # This command will clear the cache / temporary database.
 $ peg insert another_given_file.txt 
 # This command will insert another_given_file.txt into cache .
 $ peg commit -m "Commit_message"
 # This command adds all the files added using peg insert , into permanent database
 with a given commit message.
