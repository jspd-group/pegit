Help Manual -INSERT COMMAND

Name

peg-insert - Add single file, multiples files or directories in temporary database.

Syntax

peg insert [-a][--all][-i][--ignore][-v][--verbose]
[-m][--more] [specific_file_name or directory_name ]

Description

Peg insert command - This command adds the given files or content of the directory-tree into a temporary database , to prepare the contents to be staged for the next commit .After making any changes to the working directory, and before running the commit command, you must use the add command to add any new or modified files to the temporary database.
This command can be performed multiple times before a commit. It only adds the content of the specified file(s) at the time the insert command is run; if you want subsequent changes included in the next commit, then you must run peg insert again to add the new content to the index.
The peg status command can be used to obtain a summary of which files have changes that are staged for the next commit i.e. which files have been added to temporary database which are yet to be committed .

Options

  • [-a] or [--all] : This options is used to add all the files and directories of the current location into the temporary database . This option is very helpful as we need not to specify each and every file independently, rather we can use this option .
  • [-i] or [--ignore] : This option is used to ignore a given file or set of files as specified by the path after using option -i .
  • [-v] or [--verbose] : This option , when used , adds the given file , also printed the name of the added files, similar to using option -a
  • [-m] or [--more] : This option is used for displaying more output on the screen terminal .

Examples

  • Add a specified file into cache area i.e. temporary database.
    •  $ peg insert given_file.txt 
    $ peg insert -a
# This option adds all the files in current location into stagged 
temporary database.


    $ peg insert -i = "path/to/be/ignored"
# This option adds all the files in current location into stagged
    
 temporary database except for those specified by the path.
